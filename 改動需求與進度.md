## 二、家長會清冊 — 完整規格彙整

> **命名**：家委會清冊與家長會清冊為**同一份**清冊，統一稱**家長會**。檔名、標題、介面皆用「家長會」。

### 2.1 納入規則（同一份清冊，兩種來源）

| 情況 | 條件 | 處理方式 |
|------|------|----------|
| **情況一** | 假別為「**公派(家長會)**」 | **整筆**請假／代課（鐘點／日薪／半日薪皆可）完整納入家長會清冊。 |
| **情況二** | 支薪為「**半日薪**」且（「**家委支出**」打勾 **或** 假別為「**身心假**」） | **僅導師費**納入家長會清冊；**薪水本身**仍依假別入各假別清冊（公假→公假清冊、身心假→身心假清冊等）。 |

- 情況一、情況二都寫入**同一份**家長會清冊（每月一檔）。情況二在家長會清冊中只列導師費請款，不重複列整筆薪資。

### 2.2 排除規則

- **半日薪 ＋ 假別「自理」**：只納入**自理清冊**，**不**納入家長會清冊。
- **身心假 ＋ 半日薪**：導師費納入家長會清冊（無須勾選家委支出）。

### 2.3 實作時需補齊

- **假別**：新增「**公派(家長會)**」選項（如 `LeaveType` 或假別選單）。
- **家委支出**：新增「**家委支出**」欄位（打勾＝該筆導師費由家長會支出）；可放在請假單或代課明細，實作時決定層級。

### 2.4 範本與產出

- **範本**：Google 試算表工作表「**家長會清冊範本**」；複製範本、填資料、產出（與其他印領清冊相同邏輯）。
- **檔名**：`民國年年年+月月_家長會_印領清冊`。
- **標題**：例「加昌國小OOO年O月代課教師印領清冊~~【級科任教師】公假家長會」。

### 2.5 欄位對應（A2～N2）

| 欄 | 表頭 | 填入內容 |
|----|------|----------|
| A | 代課日期 | 代課日期彙總 |
| B | 代課老師 | 代課教師姓名 |
| C | 薪級 | 薪級＋(有證/無證) |
| D | 表頭動態 | 見 2.6 |
| E | 代課天數 | 代課天數（半日 0.5） |
| F | 代課節數 | 代課節數 |
| G | 代課鐘點費 | 鐘點費小計 |
| H | 請假人姓名 | 請假人 |
| I | 假別 | 假別 |
| J | 請假事由 | 請假事由 |
| K | 備註 | 備註（如半日0節） |
| L | 代導師日數 (半日導師費家長會請款) | 代導師日數（半日 0.5） |
| M | 表頭動態 | 見 2.6 |
| N | 應發金額 | 應發金額 |

### 2.6 表頭動態寫入（產出時寫入）

- **D2**：`日薪` + 換行(Alt+Enter) + `(OO日)`，OO＝當月日數（28/29/30/31）。
- **M2**：`導師費(M月OOO元/日)`，M＝月份（1～12），OOO＝當月導師費日單價（4000÷當月天數，四捨五入）。字型：「導師費」3 字為 14 號字，其餘「(M月OOO元/日)」為 10 號字。

## 五、部署到 GAS（clasp）— 換電腦繼承記錄

本節記錄「把本機 `gas/` 程式同步到 Google Apps Script 並自動部署」的設定與流程，**換到其他電腦時可依此繼承**，不必再手動貼程式碼或手動部署。

### 5.1 為什麼要用 clasp

- **clasp push**：把本機 `gas/` 資料夾內的 `.gs`、`.html`、`appsscript.json` 上傳到 GAS 專案，等同在 script.google.com 更新程式碼。
- **clasp deploy**：在 GAS 建立新的「部署版本」，讓 Web App 網址可執行最新程式（若部署選「新版本」則 push 後即生效；若選固定版本號則需 deploy 建立新版本）。
- 不必在網頁編輯器複製貼上，也不用手動點「部署 → 新版本」。

### 5.2 專案內已具備的檔案

| 檔案 | 說明 |
|------|------|
| `.clasp.json` | clasp 設定：`scriptId`（GAS 專案指令碼 ID）、`rootDir: "gas"`（只推送 `gas/`）。**換電腦後需確認 scriptId 是否正確**，必要時從 [script.google.com](https://script.google.com) → 專案設定 → 指令碼 ID 複製貼上。 |
| `gas/appsscript.json` | GAS 專案 manifest（時區、exceptionLogging 等）。clasp 規定專案須有 manifest，否則 push 會報錯。 |
| `package.json` 指令 | `gas:push`（clasp push）、`gas:deploy`（clasp deploy --description "自動部署"）、`push`（gas:push → gas:deploy → git push）。 |
| `docs/clasp設定教學.md` | 逐步教學：安裝 clasp、登入、填寫 scriptId、第一次 push、之後用 `npm run gas:push` 或 `npm run push`。 |

### 5.3 換電腦後要做的事（繼承清單）

1. **安裝依賴**  
   ```bash
   npm install
   ```
   （專案已含 `@google/clasp`，不需全域安裝。）

2. **登入 Google**  
   ```bash
   npx clasp login
   ```  
   用**與 GAS 專案相同的 Google 帳號**在瀏覽器授權。每台電腦／每次重裝後都要做一次。

3. **確認指令碼 ID**  
   開啟專案根目錄的 `.clasp.json`，確認 `scriptId` 為該 GAS 專案的指令碼 ID。若專案是從 git 拉下來且 `.clasp.json` 已 commit，通常不需改；若在新環境另建 GAS 專案，需改為新專案的指令碼 ID。

4. **（若要自動部署）開啟 Apps Script API**  
   到 [https://script.google.com/home/usersettings](https://script.google.com/home/usersettings) 啟用 **Google Apps Script API**。未啟用時 `clasp deploy` 會失敗，`clasp push` 仍可成功。

5. **（建議）Web App 部署版本設為「新版本」**  
   在 GAS：**部署** → **管理部署** → 該 Web App 部署 → **版本** 選 **「新版本」**（Head）。這樣每次 `npm run push` 後，對外網址會自動跑最新程式，不需再在 GAS 裡手動點「新版本」。

### 5.4 日常指令（換電腦後同樣適用）

| 要做的事 | 指令 |
|----------|------|
| 只把 `gas/` 同步到 GAS（不建新版本、不推 Git） | `npm run gas:push` |
| 只建立新的 GAS 部署版本 | `npm run gas:deploy` |
| **一鍵：同步 GAS + 建立新版本 + 推 Git** | `npm run push` |

詳細步驟與常見問題見 **`docs/clasp設定教學.md`**。
