# 公開頁缺額邏輯與可能問題

公開頁（`/#/public`）的缺額來自 **Firebase Firestore** 的 `publicBoard/vacancies`，與 GAS 脫鉤。以下說明資料流與常見「看不到缺額」的原因。

---

## 一、資料流邏輯

1. **後台（待聘課務清單）**
   - 只會把「已設為公開」的待聘節次放進發佈清單。
   - 操作：每筆可切換「公開中 / 已隱藏」，或使用「全部設為公開」→ 再按 **「發佈公開」**。
   - 若**沒有任何一筆是公開**，按「發佈公開」會提示「無公開項目」，**不會寫入 Firestore**。

2. **發佈時（syncToPublicBoard）**
   - 若 **Firebase 有初始化**（`db` 存在）：寫入 Firestore  
     `publicBoard` 集合 / `vacancies` 文件，欄位：  
     `{ vacancies: [ {...}, ... ], updatedAt }`，每筆都會帶 `status: '開放報名'`。
   - 若 Firebase 未啟用：改走 GAS（需設定 `gasWebAppUrl`），公開頁讀不到 GAS 的資料（公開頁只讀 Firestore）。

3. **公開頁（PublicBoard）**
   - 讀取 `doc(db, 'publicBoard', 'vacancies')`，取 `data.vacancies`。
   - 只顯示 `status === '開放報名'` 的項目（`openOnly`）。
   - 若 **Firebase 未初始化**（例如 Vercel 沒設環境變數）：`db` 為 null，會顯示「Firebase 未初始化」，不會發送任何讀取。

---

## 二、可能遇到的問題

| 狀況 | 可能原因 | 建議作法 |
|------|----------|----------|
| 畫面上寫 **「Firebase 未初始化」** | 公開頁所在環境（如 Vercel）沒有 Firebase 設定或建置時沒吃到環境變數 | 在 Vercel 專案設定 **Environment Variables**，加入 `VITE_FIREBASE_API_KEY`、`VITE_FIREBASE_AUTH_DOMAIN`、`VITE_FIREBASE_PROJECT_ID` 等（與後台相同），儲存後 **重新建置 / Redeploy** |
| **Permission denied** / **Missing permissions** | Firestore 規則未部署或規則不允許匿名讀取 | 確認 `firestore.rules` 中 `publicBoard` 為 `allow read: if true`，然後執行 `firebase deploy --only firestore:rules` |
| 沒有錯誤，但 **「目前沒有代課缺額」** | ① 從未成功發佈過，或 ② 發佈時沒有「公開」項目，或 ③ 公開頁讀到的是舊的空資料 | ① 在後台待聘清單將要公開的項目設為「公開」→ 按 **「發佈公開」**；② 確認發佈時沒有出現「無公開項目」；③ 重新整理公開頁，必要時再按一次發佈 |
| 已加「使用者網域」仍沒缺額 | Firebase **授權網域**（Auth）只影響登入；公開頁**讀** Firestore 用 `allow read: if true`，**不需登入**。讀不到多半是：沒寫入過、或公開頁的 Firebase 未初始化 | 依上表檢查：Vercel 環境變數、Firestore 規則、以及是否在後台按過「發佈公開」且當時有公開項目 |
| 後台按發佈沒報錯，但公開頁仍空 | ① 後台與公開頁是否同一個 Firebase 專案（同一組 env） ② 發佈時是否已登入（寫入規則為 `allow write: if isSignedIn()`） | 確認兩邊用同一專案；發佈時須已登入 Firebase，否則寫入會失敗（應會跳出錯誤） |

---

## 三、建議檢查順序

1. **公開頁 F12 → Console**  
   - 有「Firebase 未初始化」→ 到 Vercel 設定環境變數並 Redeploy。  
   - 有「Permission denied」→ 部署 Firestore 規則。

2. **後台**  
   - 待聘清單中至少一筆為「公開中」→ 按 **「發佈公開」**。  
   - 若出現「無公開項目」→ 先「全部設為公開」或逐筆切換為公開，再發佈。

3. **Firebase Console**  
   - Firestore → `publicBoard` → `vacancies` 文件是否存在、`vacancies` 陣列是否有資料。  
   - 若為空或不存在，表示從未成功發佈或發佈時 payload 為空。

4. **授權網域**  
   - 若公開頁有使用 Firebase Auth 登入，須在 Auth → 授權網域加入 Vercel 網址。  
   - 僅「讀取公開缺額」不需要登入，加網域無法單獨解決「沒有缺額」；仍要以上述 1～3 為主。

---

## 四、快速對照

- **寫入**：後台（已登入）→ 待聘項目設為公開 → 按「發佈公開」→ 寫入 `publicBoard/vacancies`。  
- **讀取**：公開頁 → 讀 `publicBoard/vacancies` → 只顯示 `status === '開放報名'`。  
- **看不到缺額**：多數是 (1) 公開頁 Firebase 未初始化、(2) 從未成功發佈或發佈時無公開項目、(3) Firestore 規則未部署。
