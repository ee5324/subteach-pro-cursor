# GAS 直連與產生報表 — 該怎麼做

## 一、先確認 GAS 網址沒問題（排除 404）

1. 開啟 [Google Apps Script](https://script.google.com)，進入你的專案。
2. 上方 **部署** → **管理部署作業**。
3. 在 **進行中** 找到類型為 **「網頁應用程式」** 的那一筆（若沒有，請點 **新部署** → 類型選 **網頁應用程式**，執行身分與「誰可以存取」依需求設定，對外產生請選 **任何人**）。
4. 複製該筆的 **網址**，應為：  
   `https://script.google.com/macros/s/一串ID/exec`
5. 在系統裡：登入後 **系統設定** → **連線設定 (Google Apps Script)** → 把上面網址貼到 **Web App URL** → **儲存**。

**檢查 404**：若你貼的網址對應的部署已被封存或刪除，就會 404。請確認該筆仍在「進行中」且是「網頁應用程式」。

---

## 二、直連時，從哪裡開網站會影響能否成功

- **從本機開**（例如 `npm run dev`，用 `http://localhost:5173`）：  
  直連 GAS 較有機會成功（依環境而定），可用來**確認 GAS 網址是否正確**。
- **從 Vercel 開**（例如 `https://subteach-pro-cursor.vercel.app`）：  
  瀏覽器會擋「跨網域直連」的回應（CORS），所以**在 Vercel 上直連 GAS 產生報表會失敗**，不是網址錯，是瀏覽器安全限制。

---

## 三、目前採用：直連

- 程式**一律直連 GAS**，不經 proxy。
- **從本機開**（`npm run dev` → `http://localhost:xxxx`）：產生報表可正常使用。
- **從 Vercel 開**：瀏覽器 CORS 會擋跨域直連，**在 Vercel 網址按「產生報表」會失敗**；要產生報表時請改從本機開系統操作。

---

## 四、建議順序

1. 先依 **一** 確認 GAS 網址與部署正確。
2. 要產生報表時，在本機執行 `npm run dev`，用瀏覽器開 localhost 再操作。
