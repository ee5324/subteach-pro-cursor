# GAS 直連與產生報表 — 該怎麼做

## 一、先確認 GAS 網址沒問題（排除 404）

1. 開啟 [Google Apps Script](https://script.google.com)，進入你的專案。
2. 上方 **部署** → **管理部署作業**。
3. 在 **進行中** 找到類型為 **「網頁應用程式」** 的那一筆（若沒有，請點 **新部署** → 類型選 **網頁應用程式**，執行身分與「誰可以存取」依需求設定，對外產生請選 **任何人**）。
4. 複製該筆的 **網址**，應為：  
   `https://script.google.com/macros/s/一串ID/exec`
5. 在系統裡：登入後 **系統設定** → **連線設定 (Google Apps Script)** → 把上面網址貼到 **Web App URL** → **儲存**。

**檢查 404**：若你貼的網址對應的部署已被封存或刪除，就會 404。請確認該筆仍在「進行中」且是「網頁應用程式」。

---

## 二、直連時，從哪裡開網站會影響能否成功

- **從本機開**（例如 `npm run dev`，用 `http://localhost:5173`）：  
  直連 GAS 較有機會成功（依環境而定），可用來**確認 GAS 網址是否正確**。
- **從 Vercel 開**（例如 `https://subteach-pro-cursor.vercel.app`）：  
  瀏覽器會擋「跨網域直連」的回應（CORS），所以**在 Vercel 上直連 GAS 產生報表會失敗**，不是網址錯，是瀏覽器安全限制。

---

## 三、你可以怎麼做（二選一）

### 做法 A：堅持直連 — 產生報表時改從本機開

- 日常用 Vercel 網址沒問題（Firebase、待聘、外部申請等都正常）。
- **要產生報表／代課單時**：改在**本機**跑專案（`npm run dev`），用瀏覽器開 `http://localhost:xxxx`，再點「產生報表」等，就會用直連 GAS，不受 Vercel 的 CORS 影響。

### 做法 B：在 Vercel 上也要能產生報表 — 使用 proxy（已內建）

- 若希望**在 Vercel 網址**也能按「產生報表」就成功，會改走同源 **api/gas-proxy** 代轉，避免 CORS。
- **不必在 Vercel 設環境變數**：proxy 會使用前端傳過去的 URL（即系統設定裡的 GAS Web App URL）。只要在系統設定填好 GAS 網址即可。
- 若你偏好在 Vercel 設定，可選填 **GAS_WEB_APP_URL**；未填時以系統設定的 URL 為準。

---

## 四、建議順序

1. 先依 **一** 確認 GAS 網址與部署正確（必要時在本機試一次產生報表，確認直連沒 404）。
2. 若你**可以接受「產生報表時用本機」** → 採用 **做法 A**，不需改程式。
3. 若你**一定要在 Vercel 網址產生報表** → 採用 **做法 B**，程式已內建 proxy；只要在系統設定填好 GAS URL 即可，無須在 Vercel 設環境變數。
