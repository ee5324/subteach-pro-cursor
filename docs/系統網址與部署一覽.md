## 2. 同一個 SPA 底下的主要路由（`#/...`）

| 路徑 | 誰用 | 資料大致來自 |
|------|------|----------------|
| `#/login` | 教學組登入 | Firebase Auth |
| `#/public` | 公開缺額 | Firestore `publicBoard` 等 |
| `#/apply` | 代課報名 | Firestore |
| `#/teacher-request` | 教師請假表單 | Firestore |
| `#/sub-weekly` | 代課老師週課表（手機驗證） | Firestore `publicSubstituteSchedules` |
| `#/` 及其子路徑 | 登入後後台 | Firestore + 可呼叫 GAS |

完整路由定義在 **`App.tsx`**；新增「公開頁」時**優先加路由**，不要另建第二個前端專案，以免膨脹。

## 3. Vercel Serverless（`api/`）

| 檔案 | 路徑 | 用途 |
|------|------|------|
| `api/gas-proxy.js` | `/api/gas-proxy` | 瀏覽器經 Vercel 代打 GAS（避 CORS 等）。 |
| `api/line-webhook.js` | `/api/line-webhook` | LINE → 先回 200 → 背景轉 GAS。 |

環境變數（Vercel 專案設定）：

- **`GAS_WEB_APP_URL`**：GAS `.../exec`（給上述兩支 API 用）。
- 前端建置仍用 **`VITE_*`**（例如 `VITE_GAS_WEB_APP_URL`），與本機 `.env` 對應。

## 5. 程式防膨脹（實務原則）

1. **網址常數**：GAS 預設只維護 **`config.ts`**（與 Cursor 規則中的部署 ID）；Vercel 用環境變數，不硬編在業務元件裡。  
2. **新功能**：能進 **`pages/` + 路由** 就不要新開 repo 或第二個 deploy target。  
3. **LINE / 除錯**：GAS 的 `LINE_NOTIFY_*` 屬性測完即關；正式環境不要長期開 `LINE_NOTIFY_DEBUG_WEBHOOK`。  
4. **Cloud Functions**：本 repo 未納入 Firebase Functions 目錄（減少未使用腳手架）；若日後需要，再於專案根新增 `functions/` 並在 `firebase.json` 設定 `functions.source`。  
5. **大檔**：`SheetManager.gs` 等已很大；新邏輯優先 **獨立 `.gs` / 模組** 再在 `Controller` 接線，避免單檔無限長。

---

## 6. 與 Cursor 規則的關係

- **GAS 部署**：仍請自行在 GAS／clasp 操作，Agent 不代為 `clasp push/deploy`（見 `.cursor/rules/gas-deployment.mdc`）。  
- **變更 GAS 網址時**：更新 `config.ts`、本表、以及規則／部署文件中的同一 URL，**一處一來源**。
