## 2. 維護者建議具備的權限

- **GitHub**：此 repo 的讀寫（或至少能發 PR、合併至部署分支）。
- **Vercel**（或實際託管前端之平台）：環境變數、網域、部署紀錄。
- **Firebase 專案**（見 repo 內 `.firebaserc`）：Authentication、Firestore（規則與索引）、必要時 Storage。
- **Google**：綁定 GAS 的試算表／Apps Script 專案、Drive 輸出資料夾（依各校實際設定）。
- **各校行政約定**：誰能改「白名單」、誰能解鎖段考提報等（見下節）。

## 3. 每學期更新：原則與實務清單

以下「每學期」指：**新學年或新學期**開始前後，資料與設定會隨班級、導師、節次、名單變動而需調整。系統**沒有**單一按鈕「全自動新學期重置」；請依模組**分項處理**。

### 3.1 代課主系統（請假、代課、薪資、清冊）

| 項目 | 建議時機 | 作法概要 |
|------|----------|----------|
| **學期起訖、畢業日、假日** | 開學前 | **系統設定**頁更新；影響自動計算與日期合理性。 |
| **教師課表／任課班級** | 課表確定後 | **教師管理**等頁面更新；與代課派任、統計相關。 |
| **Firestore 白名單** | 人事或帳號變動時 | 依各校流程更新可登入者；詳見專案內設定與 `firestore.rules`。 |
| **需 GAS 的匯出／報表** | 更換 GAS 部署網址時 | 同步更新 **設定中的 GAS URL** 與 Vercel／建置用的環境變數；見 `config.ts` 與部署一覽。 |

**原則**：日常增刪改代課資料在 **Firebase** 即可完成；**產生試算表／特定報表**才依賴 **GAS**（見 `資料存放與GAS功能說明.md`）。

### 3.2 教學組事務：語言選修名單（段考填報會用到）

- 學生名單依 **學年度** 分開存放（教學組「學生名單」／匯入 Excel 流程，見 `edutrack/README.md`）。
- **新學年**通常需要：**新學年名單**（上傳或合併匯入），與前一年資料分開；合併規則可避免「整份刪光再匯」，但**仍須提供反映新分班結果的來源檔**。
- **原則**：名單是學年維度；學期內若僅微調，在該學年文件內編修即可。

### 3.3 段考名單填報（優異／進步等提報）

| 項目 | 說明 |
|------|------|
| **段考活動** | 每學期／每次段考應建立**新的段考活動**（含學年等欄位），與舊活動分開，避免資料混在一起。 |
| **導師白名單** | 導師 Google 帳號多可**沿用**；若僅換班級，在管理端**修改該筆的班級／備註**即可，**不必**整批刪除重匯帳號。 |
| **填報畫面班級選單** | 來自該次活動所綁 **學年** 的語言選修名單；新學年請先確認名單已更新。 |
| **解鎖** | 已鎖定之提報需修改時，由管理端依流程**解鎖**後再請導師重送。 |

**原則**：「人（Email）」盡量穩定；「班級、活動、學年」隨學期更新。

### 3.4 學期轉換時的風險控管

- 變更 **學期日期** 前，先確認是否影響進行中的請假／代課紀錄。
- 新建 **段考活動** 時，確認 **學年** 與名單一致，避免導師選到舊名單。
- **GAS** 若需改版：先在測試部署驗證，再替換正式 Web App URL，並保留上一版備份（各校自訂流程）。

## 4. 前端與部署（Vercel）

- 路由定義於根目錄 **`App.tsx`**（HashRouter）；公開路徑例如 `#/exam-submit`（段考填報）等。
- **環境變數**：正式站以 Vercel 設定為準；本機開發用 `.env`（勿將金鑰提交至公開 repo）。
- **建置與部署**：依團隊慣例（例如 `npm run build` 後由 Vercel 連動 Git）；細節見 `docs/系統網址與部署一覽.md`。

---

## 5. Google Apps Script（GAS）

- 原始碼在 repo 的 **`gas/`**，實際執行在 **Google Apps Script 專案**（與試算表綁定）。
- **部署與 push**：請由維護者在 **GAS 網頁或本機 clasp** 自行操作；請勿依賴未經審核的自動覆蓋正式腳本（見 `.cursor/rules/gas-deployment.mdc`）。
- 變更 **Web App URL** 時：必須同步更新專案設定與文件中的同一 URL，避免前端呼叫舊網址。

---

## 6. Firebase

- **Firestore 規則**：`firestore.rules`（根目錄）；教學組另有 `edutrack/firestore.rules`，部署前應確認專案使用哪一份與如何合併。
- 變更規則後需 **部署規則**；索引需求若變更，需處理 `firestore.indexes.json` 等（見 Firebase 主控台錯誤提示）。
- **備份／匯出**：依學校資安政策決定是否定期匯出 Firestore 或僅依賴 Google 備份機制。

## 7. 除錯捷徑

| 現象 | 可檢查方向 |
|------|------------|
| 無法登入 | Firebase Auth、白名單集合、帳號是否啟用 |
| 匯出／報表失敗 | `GAS_WEB_APP_URL`、GAS 部署權限（Anyone）、GAS 端錯誤記錄 |
| 教學組資料讀不到 | Firestore 規則、`edutrack_allowed_users`、是否登入正確帳號 |
| 段考填報無法送出 | 導師是否在 `exam_submit_allowed_users`、活動是否鎖定、規則是否允許 update |
| 段考填報網址錯誤 | 應為 **`origin + pathname + #/exam-submit`**（子路徑部署勿用根目錄硬編） |

## 8. 程式庫習慣（給日後改程式的人）

- 新功能優先放在既有 **`pages/`** 與路由，避免另開第二個前端專案。
- **網址常數**：GAS 預設集中於 **`config.ts`** 與環境變數，不要在各元件散落硬編。
- **教學組**：API 集中 **`edutrack/services/api.ts`**；型別 **`edutrack/types.ts`**。
- **大檔**：`gas/SheetManager.gs` 等已偏大，新邏輯優先獨立新檔再在 Controller 接線（見部署一覽「程式防膨脹」一節）。
