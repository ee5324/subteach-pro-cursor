# 公開缺額報名 — 審核完畢後如何通知代課老師

## 現況

- **公開招聘頁面**（`#/public`）：代課老師選擇缺額後填寫姓名、電話、備註並送出，資料寫入 Firestore `publicBoardApplications`（一筆報名對應一個缺額 ID）。
- **管理員審核**：目前是在 **待聘清單**（`#/pending`）中，為每個待聘節次**手動指派代課老師**；指派完成即等同「審核完畢」。
- **通知**：**目前系統沒有自動通知功能**。審核完畢（派代完成）後，代課老師**不會**收到系統發送的簡訊、Email 或站內訊息，需由管理員自行以電話／簡訊／Email 等方式聯絡報名者。

---

## 老師目前如何「接到通知」？

| 做法 | 說明 |
|------|------|
| **管理員主動聯絡** | 在待聘清單指派該老師後，由管理員依報名時留下的**電話**或**備註**中的聯絡方式，以電話／Line／Email 通知對方已排入或已派代。 |
| **公開頁說明** | 公開缺額公告頁面已有「系統將儘快與您聯繫」等文案，可再加一句：「審核結果將由校方以電話或簡訊通知，請保持聯絡方式暢通。」 |

---

## 自動通知：成本最低、設置最少的方式

| 方式 | 成本 | 學校端設置 | 老師端步驟 | 說明 |
|------|------|------------|------------|------|
| **Email** | 免費（Gmail 或 SendGrid 免費額度） | 表單加「Email」欄位；1 支 Cloud Function 或 Firebase Extension（Trigger Email） | 填寫 Email 即可 | 實作與維護最單純，多數人都有信箱。 |
| **LINE Notify（權杖）** | 免費（**已宣布 2025/3/31 終止服務**） | 表單加「LINE 通知權杖」選填；1 支 Cloud Function 呼叫 LINE Notify API | 老師需至 LINE Notify 取得權杖並貼回表單 | **不建議新採用**：官方將於 2025 年 3 月 31 日結束服務，請改用 Messaging API。 |
| **LINE Messaging API（官方 Bot）** | 免費（有訊息額度） | 申請 LINE 開發者、建立 Messaging API 頻道、表單或綁定頁做 LINE Login、1 支 Cloud Function 發送 | 用 LINE 登入／加 Bot 一次即可 | 老師體驗最好，但學校要申請與設定頻道、網域等。 |
| **簡訊（SMS）** | 依則數計費 | 串接 Twilio／三竹等、1 支 Cloud Function | 留手機即可 | 到達率高，但需付費且要簽約／金流。 |

**結論：成本與設置都最少的是 Email，不是 LINE。**

- **推薦優先：Email**  
  表單多一個「Email」欄位，派代完成時用 **Firebase Cloud Function** 觸發寄信（或 Firebase Extension「Trigger Email」＋ SendGrid 免費額度）。不需 LINE 開發者、不需簡訊業者，老師也只多填一欄。
- **若一定要用 LINE**  
  - **LINE Notify（權杖）**：**已宣布 2025 年 3 月 31 日終止服務**，不建議新採用。  
  - **建議**：**LINE Messaging API**——學校需申請 LINE 開發者與頻道、做 LINE Login／綁定，老師只要用 LINE 登入或加 Bot 一次即可收到通知，且為官方建議的替代方案。

## LINE 通知：成本與難度 × 代課老師最少步驟

你希望代課老師**最少步驟**就能用 LINE 收到通知，需要先區分兩種做法：

### 做法 A：LINE Notify（權杖）— **已宣布終止，不建議新採用**

| 項目 | 說明 |
|------|------|
| **維護狀況** | **LINE 官方已宣布 LINE Notify 將於 2025 年 3 月 31 日結束服務**（[官方公告](https://developers.line.biz/en/news/2024/10/07/line-notify-will-be-discontinued/)）。屆時 API、權杖發行、官網等皆停止。新專案請改用 **LINE Messaging API**。 |
| **成本** | 完全免費，無則數限制（至 2025/3/31 止）。 |
| **學校端難度** | 低。不需申請 LINE 開發者或 Bot，只要表單加「LINE 通知權杖」選填 ＋ Cloud Function 呼叫 LINE Notify API。 |
| **代課老師要做的事** | ① 打開 LINE Notify → ② 登入 LINE → ③ 發行權杖 → ④ 複製權杖 → ⑤ 回到報名表單貼上。 |
| **問題** | 老師步驟多、且**服務即將終止**，**不符合「最少步驟」且不適合長期使用**。 |

### 做法 B：LINE Messaging API（官方 Bot + LINE Login）

| 項目 | 說明 |
|------|------|
| **成本** | 免費（依 LINE 方案有訊息則數額度，一般校方通知用量足夠）。 |
| **學校端難度** | 較高。要申請 [LINE 開發者](https://developers.line.biz/)帳號、建立 **Messaging API** 頻道、在報名頁做 **LINE Login**（一鍵綁定）、派代完成時用 Cloud Function 呼叫 Messaging API 發送 Push 訊息。 |
| **代課老師要做的事** | ① 在報名頁點「用 LINE 接收通知」→ ② 在 LINE 裡按「同意」→ ③ 回到報名頁填姓名／電話／選缺額 → ④ 送出。之後有派代就會自動收到 LINE 訊息。 |
| **優點** | 老師只做 **1～2 步**（點綁定 + 同意），不用碰權杖、不用離開 LINE 官網，**最符合「最少步驟」**。 |

### 結論與建議

- **LINE Notify 將於 2025 年 3 月 31 日終止服務**，新功能不建議再以 LINE Notify 權杖實作；既有使用者也應規劃遷移至 **LINE Messaging API**。
- **若目標是「代課老師最少步驟就用 LINE 收到通知」**：應採用 **做法 B（LINE Messaging API + LINE Login）**。  
  - 老師端：報名時按一次「用 LINE 接收通知」並在 LINE 授權，之後就會用 LINE 收通知。  
  - 學校端：需一次性的 LINE 開發者與頻道設定、以及報名頁的 LINE Login 與一支 Cloud Function。
- **不建議新採用做法 A（LINE Notify 權杖）**：除老師端步驟多外，服務即將結束，不利長期維護。

若要實作「做法 B（老師最少步驟 + LINE 通知）」，需要：1) LINE 開發者帳號與 Messaging API 頻道；2) 報名表單上的「用 LINE 綁定」按鈕（LINE Login）；3) 將登入後取得的 LINE userId 與報名資料一併存進 Firestore；4) 派代完成時 Cloud Function 依 userId 呼叫 Messaging API 發送訊息。我可以依你現有專案結構，列出具體步驟與程式修改草稿（含 Firestore 欄位、Cloud Function 範例）。

## 若未來要「系統自動通知」（實作選項）

1. **Email 通知**
   - 管理員於待聘清單「指派代課老師」時，若該缺額的報名資料有留 Email，由 Cloud Function 或 Trigger Email 寄出「您已獲派 O/O 節 OO 科目」等內容。
   - 需：報名表單增加「Email」欄位並寫入 Firestore；部署 1 支 Cloud Function（或安裝 Firebase Extension ＋ 設定 SMTP／SendGrid）。

2. **LINE Notify（權杖）** — **不建議：將於 2025/3/31 終止服務**
   - 報名表單選填「LINE 通知權杖」；派代完成時 Cloud Function 以該權杖呼叫 LINE Notify API。官方已公告結束服務，請改用 Messaging API。

3. **LINE Messaging API**
   - 報名或綁定時以 LINE Login 取得老師的 LINE userId；派代完成時 Cloud Function 呼叫 Messaging API 推播。
   - 需：LINE 開發者帳號、Messaging API 頻道、LINE Login 或綁定流程、1 支 Cloud Function。

4. **簡訊（SMS）**
   - 派代完成時由 Cloud Function 呼叫 Twilio／三竹等 API 發送至報名手機。
   - 需：簡訊業者帳號、付費、Cloud Function。

5. **一鍵複製／mailto（半自動）**
   - 待聘清單或報名紀錄畫面提供「複製電話」或「寄信給報名者」（mailto），方便管理員手動通知。
   - 不需後端寄信或 LINE，實作快，但仍是管理員手動通知。

---

## 建議

- **短期**：維持現狀，審核完畢後由**管理員依報名資料（電話等）主動聯絡**代課老師；並在公開頁說明「審核結果將由校方以電話或簡訊聯繫」。
- **自動通知、成本與設置最少**：優先做 **Email**（表單加 Email ＋ 派代完成時 Cloud Function 或 Trigger Email 寄信）。
- **若堅持用 LINE**：**請用 LINE Messaging API**。LINE Notify 將於 2025/3/31 終止，不宜新採用。
