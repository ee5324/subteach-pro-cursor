# 寫入流程確認（家委支出、公派(家長會)）

## 1. 寫入路徑是否正確

### 建立/編輯代課單（EntryForm）
- **假別**：`leaveType` 含 `公派(家長會)`，從選單選取後寫入 `recordData.leaveType`。
- **家委支出**：`homeroomFeeByPta` 從 checkbox 寫入 `recordData.homeroomFeeByPta`，已改為**一律寫入布林值**（`!!homeroomFeeByPta`），未勾選會寫入 `false`，避免舊的 `true` 殘留。
- **儲存**：`recordData` 整筆傳給 `addRecord(recordData)` 或 `updateRecord(recordData)`。

### 待辦批次派代（PendingItems）
- 批次套用時代入 `homeroomFeeByPta: bulkHomeroomFeeByPta`，並呼叫 `updateRecord({ ...record, slots, details, homeroomFeeByPta })`，會一併寫回 Firestore。

### Firestore 儲存（AppContext）
- **addRecord**：`setDoc(doc(db, 'records', record.id), record)`，整份 document 寫入，包含 `leaveType`、`homeroomFeeByPta`。
- **updateRecord**：改為 `setDoc(docRef, { ...updatedRecord }, { merge: true })`，以合併方式寫入，確保 `homeroomFeeByPta`、`leaveType` 等欄位都會被更新。

### 讀取
- `onSnapshot(collection(db, 'records'), ...)` 用 `d.data() as LeaveRecord`，會把 Firestore 裡的 `leaveType`、`homeroomFeeByPta` 讀進 state，介面與 GAS 用的都是同一份資料。

### 產出清冊（GAS）
- 前端呼叫 `GENERATE_REPORTS` 時傳入 `records: filteredRecords`，即目前畫面上篩選後的 `records`（來自 Firestore）。
- GAS 使用 `record.leaveType === '公派(家長會)'`、`record.homeroomFeeByPta` 判斷家長會清冊的「情況一」「情況二」。

---

## 2. 已做的修正

1. **EntryForm**：`homeroomFeeByPta: homeroomFeeByPta || undefined` → `homeroomFeeByPta: !!homeroomFeeByPta`，未勾選時明確寫入 `false`。
2. **AppContext updateRecord**：由 `updateDoc(ref, updatedRecord)` 改為 `setDoc(ref, { ...updatedRecord }, { merge: true })`，避免漏寫欄位，並確保 `homeroomFeeByPta`、`leaveType` 等都會被正確寫入。

---

## 3. 如何自行確認

1. **假別**：建立一筆假別「公派(家長會)」的代課單 → 儲存 → 重新整理或重開頁面 → 編輯該筆，確認假別仍為「公派(家長會)」。
2. **家委支出**：建立一筆半日薪並勾選「家委支出」→ 儲存 → 重新整理 → 再開該筆或到待辦列表，確認仍為勾選狀態；取消勾選儲存後再確認變為未勾選。
3. **清冊**：對有「公派(家長會)」或「半日薪+家委支出」的紀錄產出印領清冊，檢查家長會清冊是否有對應列與金額。

若以上都符合，即可確認目前能正確寫入並用於清冊。
