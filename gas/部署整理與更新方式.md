## 一、把多筆部署整理乾淨（只做一次）

GAS 無法「刪除」部署，只能**封存**。封存後會移到「已封存」，畫面上就只會剩一筆「進行中」。

### 步驟

1. 開啟 [Google Apps Script](https://script.google.com) → 開啟本專案的 GAS 專案。
2. 上方選單點 **部署** → **管理部署作業**。
3. 在 **進行中** 區塊：
   - **保留一筆**：部署作業 ID 為上表 `AKfycbzZ2C-mQkzNjfkixHupdU_...`、網址為上表網頁應用程式 URL、且已設「任何人」可存取的那一筆（若名稱是「未命名」或「最新程式」沒關係，認 ID 或網址即可）。
   - **其餘每一筆**：點該筆右側 **⋮（三個點）** → 選 **封存**。
4. 封存後，多餘的會移到 **已封存**，「進行中」只會剩您要用的那一筆。

## 二、之後更新程式（維持同一筆部署、不新增）

之後改程式請**不要**再按「新部署」，改為**只更新現有那一筆的版本**，這樣網址不變，也不會多出一堆部署。

### 步驟

1. **推送程式碼**（本機專案目錄）：
   ```bash
   npx clasp push --force
   ```
2. 到 GAS：**部署** → **管理部署作業**。
3. 點選您保留的那一筆（進行中）右側 **鉛筆圖示（編輯）**。
4. 在 **版本** 下拉選單：
   - 選 **標頭**（代表永遠用最新程式），或  
   - 選剛 push 後出現的**最新版本編號**。
5. 按 **部署** 儲存。

之後前端與 Vercel 的 `GAS_WEB_APP_URL` 都不用改，沿用同一個網址即可。

## 三、若之後需要「新部署」（例如要換網址時）

1. **先封存舊的**：到 **部署** → **管理部署作業**，將目前使用中的那一筆點 ⋮ → **封存**，避免「進行中」累積多筆。
2. **再建立新部署**：點 **新部署**，**類型** 請選 **網頁應用程式**（不要選到「API 執行」等），並設定：
   - **說明**：可自訂名稱（例如「最新程式」）。
   - **執行身分**：依需求選「我」或「存取網頁應用程式的使用者」。
   - **誰可以存取**：對外給表單／proxy 呼叫時請選 **任何人**。
3. 部署完成後，複製新的 **網頁應用程式 URL**，更新專案 `config.ts` 的 `GAS_WEB_APP_URL` 與 Vercel 的 `GAS_WEB_APP_URL`。

**重點**：新部署時一定要把**類型**選成 **網頁應用程式**，才能得到可對外開啟的 exec 網址。

---

## 四、若只是不小心多出一筆部署

到 **部署** → **管理部署作業**，把多出來的筆數在「進行中」那區封存掉，只留正在用的那一筆即可。
